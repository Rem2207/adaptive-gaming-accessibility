# Project Rules

## General
- Maintain modular architecture.
- Do not rewrite the entire project unnecessarily.
- Prefer incremental safe changes.
- Keep code readable and professional.

## Technology
- Python project.
- Tkinter UI.
- OpenCV / MediaPipe supported.
- JSON for settings persistence.

## Functional Goals
- Accessibility-first design.
- Support motor disability users.
- Support hearing disability users.
- Adaptive input systems.

## Coding Rules
- Use classes where appropriate.
- Use type hints.
- Use dataclasses when useful.
- Separate UI / services / config.

## Important
- Preserve working MVP features.
- Avoid breaking current gesture recognition.
- Avoid removing existing modules.

## Style
- Use English for code.
- Use Spanish comments only when necessary.

## Accessibility & Localization
- Provide English and Spanish UI support.
- Use clear and simple wording.
- Keep labels readable.
- Future-ready for more languages.
